Як правильно "натягнути" верстку на WordPress: повний гайд

📅 12 березня 2025 ✍️ Іван Іваненко 📁 Технології ⏱ 8 хв читання 💬 8 коментарів
🖼

Щоразу коли ви беретеся за новий проєкт на WordPress, виникає одне й те саме питання: з чого почати перетворення готового HTML/CSS на повноцінну тему? У цьому гайді ми розберемо процес крок за кроком.

Крок 1: Структура файлів теми

Мінімально необхідні файли для будь-якої теми WordPress — це лише два: style.css та index.php. Але для повноцінної роботи вам знадобиться більше:

  • style.css — головний CSS + метадані теми
  • index.php — шаблон за замовчуванням
  • header.php — шапка сайту
  • footer.php — підвал сайту
  • sidebar.php — бічна панель
  • single.php — окремий пост
  • page.php — статична сторінка
  • archive.php — архів / категорія
  • functions.php — реєстрація хуків, меню, сайдбарів

Крок 2: Розрізаємо HTML на PHP-шматки

Ваш index.html треба розрізати на частини. Все, що знаходиться до <main> — іде у header.php. Після закриваючого </main> — у footer.php.

💡 Ключове правило: WordPress збирає сторінку з шаблонних файлів через функції get_header(), get_footer(), get_sidebar().

Крок 3: The Loop

Серце WordPress — це "The Loop". Вся виведення постів відбувається через нього:

<?php if ( have_posts() ) : ?>
  <?php while ( have_posts() ) : the_post(); ?>

    <article id="post-<?php the_ID(); ?>" <?php post_class(); ?>>
      <h2><a href="<?php the_permalink(); ?>"><?php the_title(); ?></a></h2>
      <?php the_excerpt(); ?>
    </article>

  <?php endwhile; ?>
<?php endif; ?>

Крок 4: Реєстрація меню та сайдбарів

У файлі functions.php потрібно зареєструвати всі зони меню та сайдбарів, щоб вони з'явилися в адмінпанелі WordPress:

function wplearn_setup() {
    register_nav_menus([
        'primary' => 'Головне меню',
        'footer'  => 'Меню у футері',
    ]);
}
add_action( 'after_setup_theme', 'wplearn_setup' );

function wplearn_widgets_init() {
    register_sidebar([
        'name'          => 'Бічна панель',
        'id'            => 'sidebar-1',
        'before_widget' => '<div class="widget">',
        'after_widget'  => '</div>',
        'before_title'  => '<h3 class="widget__title">',
        'after_title'   => '</h3>',
    ]);
}
add_action( 'widgets_init', 'wplearn_widgets_init' );

Тепер меню та сайдбари з'являться у Зовнішній вигляд → Меню та Зовнішній вигляд → Віджети в адмінпанелі.

Підсумок

Процес "натягування" верстки на WordPress — це в першу чергу розуміння ієрархії шаблонів і правильне використання WordPress API. Не поспішайте: розберіть кожну функцію, прочитайте документацію на developer.wordpress.org.
